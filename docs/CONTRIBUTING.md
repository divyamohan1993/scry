The following is a set of guidelines for contributing to Scry.

## Code of Conduct

This project and everyone participating in it is governed by a Code of Conduct. By participating, you are expected to uphold this code.

## How Can I Contribute?

### Reporting Bugs

This section guides you through submitting a bug report for Scry. Following these guidelines helps maintainers and the community understand your report, reproduce the behavior, and find related reports.

*   **Use the issue tracker.** warning us about a potential bug.
*   **Describe the bug.** clear and concise description.
*   **Reproduce the bug.** steps to reproduce.

### Suggesting Enhancements

This section guides you through submitting an enhancement suggestion for Scry, including completely new features and minor improvements to existing functionality.

*   **Use the issue tracker.**
*   **Detail your suggestion.**

### Your First Code Contribution

1.  Fork the repository.
2.  Clone the fork to your local machine.
3.  Create a new branch for your feature or fix.
4.  Make your changes.
5.  Run tests: `pytest`
6.  Ensure no secrets are committed!
7.  Push your changes and open a Pull Request.

## Style Guide

*   Use `flake8` for linting.
*   Follow PEP 8 conventions for Python code.
*   Write clear, commented code where necessary.
